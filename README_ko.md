질문이나 예측을 제시하면 OpenPE가 제1원리에서 종국까지 추적합니다 — 인과 체인을 구축하고, 모든 연결을 반박 검증으로 테스트하며, 증거가 소진될 때까지 전방으로 투영합니다. 반환되는 결과: 모든 엣지에 정량화된 신뢰도를 가진 인과 DAG, 모든 인과 주장에 대한 3중 반박 검증, 견고함 / 분기 의존 / 균형 / 불안정으로 분류된 시나리오 예측, 그리고 모든 발견을 데이터 출처에 연결하는 기계 검증 가능한 감사 추적.

전체 파이프라인 — 가설 생성, 데이터 수집, 인과 검증, 예측, 보고서 — 이 자율적으로 실행됩니다. 기본 원리: 설명력은 인과 체인을 따라 곱셈적으로 감쇠하며, 프레임워크는 그 감쇠를 사용하여 무엇을 조사할 가치가 있는지, 언제 멈출지를 결정합니다.

## 작동 방식

```
질문 ─→ 0단계: 발견 ─→ 1단계: 전략 ─→ 2단계: 탐색
          │              │              │
      인과 DAG        방법 선택       EDA + 차트
      데이터 수집     EP 평가        분포 확인
      품질 게이트     체인 계획
          │              │              │
          ▼              ▼              ▼
      3단계: 분석 ─→ 4단계: 예측 ─→ 5단계: 검증
          │              │              │
      인과 검증      몬테카를로 시뮬레이션  독립 재현
      3중 반박 배터리  민감도 분석       EP 전파 감사
      EP 전파        종국 분류         인과 라벨 감사
          │              │              │
          ▼              ▼              ▼
                  6단계: 문서화
                      │
                  분석 보고서 (MD + PDF)
                  감사 추적 (주장 + 출처 + 논리)
                  EP 감쇠 시각화
```

각 단계는 동일한 루프를 따릅니다:

1. **실행** — 서브에이전트가 산출물 생성 (계획 모드로 시작)
2. **검토** — 독립 검토자가 이슈를 A(차단) / B(수정 필요) / C(제안)으로 분류
3. **반복** — 구조화된 수정 사항 적용, 중재자가 범위 지정 diff로 검증 (전체 재검토 아님)
4. **커밋** — STATE.md에 상태 추적
5. **인간 게이트** — 5단계 이후, 최종 보고서 전 인간 승인

## 네 가지 핵심 혁신

### 1. 설명력 (EP) — 사전 분포에서 사후 분포까지의 전체 생명주기

대부분의 분석 프레임워크는 신뢰도를 이진적으로 처리합니다: "믿는다" 또는 "안 믿는다." OpenPE는 **설명력 (Explanatory Power)** 을 도입합니다 — 인과 체인을 따라 얼마나 많은 설명 가치가 남아있는지 추적하고, 전체 분석 생명주기에 걸쳐 진화하는 연속적이고 곱셈적인 척도입니다.

**공식:**
```
EP = truth × relevance

truth:     이 인과 관계가 실제인지에 대한 확신 (0–1)
relevance: 이 인과 관계가 결과의 분산을 얼마나 설명하는지 (0–1)
```

**EP는 인과 체인을 따라 곱셈적으로 감쇠합니다.** A → B → C → D의 경우:
```
Joint_EP = EP(A→B) × EP(B→C) × EP(C→D)
```

각 엣지가 EP=0.7인 5링크 체인의 Joint_EP는 0.17 — 소프트 절단 임계값을 간신히 넘는 수준입니다. 이는 프레임워크의 한계가 아니라, 간접 인과 관계에서 설명력이 소산되는 방식을 정직하게 표현한 것입니다.

**절단 임계값이 중단 규칙을 강제합니다:**

| Joint EP | 행동 |
|----------|------|
| < 0.05 | **하드 절단** — 중단. 이 체인은 분석 지평선을 넘었습니다. |
| 0.05–0.15 | **소프트 절단** — 경량 평가만. 서브체인 확장 없음. |
| > 0.30 | **서브체인 확장** — 상세 조사 가치 있음. 서브분석 생성. |

**EP는 분석을 통해 진화합니다.** 증거가 축적되면서 라벨과 truth 값이 변화합니다:

| 단계 | 분류 | truth | 의미 |
|------|------|-------|------|
| 분석 전 (사전) | `LITERATURE_SUPPORTED` | 0.70 | 기존 문헌에서 확립됨 |
| 분석 전 (사전) | `THEORIZED` | 0.40 | 그럴듯한 메커니즘, 미검증 |
| 분석 전 (사전) | `SPECULATIVE` | 0.15 | 가능하지만 근거 취약 |
| 3단계 후 (사후) | `DATA_SUPPORTED` | 0.85 | 3중 반박 배터리 통과 |
| 3단계 후 (사후) | `CORRELATION` | 0.50 | 통계적 신호 존재, 인과성 미확립 |
| 3단계 후 (사후) | `HYPOTHESIZED` | 0.15 | 현재 데이터로 검증 불가 |
| 3단계 후 (사후) | `DISPUTED` | 0.30 | 증거가 내부적으로 모순됨 |

사전-사후 전환은 기계적이며 재량적이지 않습니다: 분류 표가 `truth` 값으로 직접 매핑되고, DATA_SUPPORTED 엣지의 `truth`는 `min(1.0, max(0.8, prior_truth + 0.2))`로 제한됩니다. 어떤 분석가도 이 산술을 무시할 수 없습니다.

**DISPUTED 분류는 반박 배터리의 내부 모순을 탐지합니다.** 모든 인과 엣지는 세 가지 검증을 거칩니다: 위약 검증(처리 변수를 랜덤 변수로 대체; 효과가 사라져야 함), 랜덤 공통 원인 검증(랜덤 혼란 변수 추가; 추정값이 안정적이어야 함), 데이터 부분집합 검증(랜덤 80% 부분집합으로 추정; 일관적이어야 함). 위약 결과가 인과적 앵커 역할을 합니다:

- 위약이 **통과**하면 (효과가 처리 특이적), 부분집합이나 공통원인 검증 실패는 모순 — 실제 효과는 안정적이어야 합니다
- 위약이 **실패**하면 (효과가 처리 특이적이지 않음), 부분집합 통과는 모순 — 비실제 효과는 완벽하게 안정적일 수 없습니다

```python
# 모순 탐지 로직 (Scheme C)
if placebo_passed and (not subset_passed or not cc_passed):
    return "DISPUTED"   # 실제이지만 불안정 — 모순
if not placebo_passed and subset_passed:
    return "DISPUTED"   # 비실제이지만 완벽히 안정 — 모순
```

DISPUTED 엣지는 알고리즘적 추측 대신 인간 검토로 라우팅됩니다. 프레임워크는 모순된 증거를 자동 분류하기를 거부합니다.

**왜 중요한가:** 모든 인과 논증은 산문 속에 불확실성을 숨기는 대신 독자가 추적하고 도전할 수 있는 숫자를 가집니다. EP 감쇠 차트 — 모든 OpenPE 보고서의 핵심 그림 — 는 확립된 발견에서 전방 예측까지 이 전파를 시각화하여 인식론적 정직함을 눈에 보이게 합니다.

### 2. 분석 지평선 — 자율 연구 경계로서의 EP 절단

대부분의 분석 프레임워크는 사용자가 제공하는 어떤 인과 구조에 대해서든 신뢰도 점수를 계산합니다. OpenPE는 다르게 합니다: EP를 사용하여 **무엇을 조사할 가치가 있는지**를 결정합니다. 분석 지평선은 Joint_EP의 곱셈적 감쇠가 추가 조사를 인식론적으로 정당화하지 못하는 지점입니다.

이 구분은 실제로 매우 중요합니다. 중단 규칙이 없는 자율 에이전트는 원인을 무한정 추적합니다 — 모든 결과에는 원인이 있고, 모든 원인에는 더 이전의 원인이 있으며, 체인은 끝나지 않습니다. 분석 지평선은 에이전트에게 임의적인 깊이 제한이 아닌, 증거 구조에 근거한 원칙적인 중단 이유를 제공합니다.

**세 가지 하드 임계값이 연구 경계를 결정합니다:**

| Joint EP | 상태 | 행동 |
|----------|------|------|
| > 0.30 | 조사 가치 있음 | 서브분석 생성; 완전한 인과 검증 필요 |
| 0.15–0.30 | 한계 영역 | 경량 평가만; 문서화하되 확장 금지 |
| 0.05–0.15 | 소프트 절단 | 간략 메모; 서브체인 확장 불허 |
| < 0.05 | 하드 절단 | 중단. 이 체인은 분석 지평선을 넘었습니다. |

**지평선은 창발적이며, 미리 정해진 것이 아닙니다.** 증거가 소진되는 곳에서 결정됩니다 — 파라미터가 지정하는 곳이 아닙니다. 두 개의 강한 엣지(EP=0.85)를 가진 인과 체인은 다섯 개의 약한 엣지를 가진 체인보다 더 멀리 뻗어나갑니다. 깊이는 증거의 결과이지, 설정 선택이 아닙니다.

**이 설계 결정은 프로젝트 이름에서 직접 비롯됩니다.** "원리에서 종국으로"는 은유가 아닙니다 — 분석 계약의 문자 그대로의 설명입니다. 프레임워크는 제1원리 인과 가설에서 시작하여 종단 상태(종국)나 분석 지평선(증거가 소진되는 곳)에 도달할 때까지 앞으로 진행합니다. 언제 멈출지 결정하는 데 인간의 판단이 필요하지 않습니다; EP 산술이 그 결정을 내립니다.

**지평선은 데이터 품질과 상호작용합니다.** 데이터 소스가 LOW 품질 판정을 받으면, 그것이 지지하는 엣지들이 관련성 페널티를 받습니다. 이는 임계값에 가까운 체인을 소프트 절단 경계를 넘어 끌어당길 수 있습니다 — 데이터 품질이 저하됨에 따라 분석 범위가 *자동으로 축소*됩니다. 저품질 데이터셋은 실패를 유발하지 않습니다; 프레임워크가 얼마나 멀리 볼 수 있는지에 대해 정직하게 만듭니다.

**왜 중요한가:** 중단 규칙이 없는 분석 시스템은 분석 시스템이 아닙니다 — 무한정 작업을 생산하는 연구 생성기입니다. 분석 지평선은 "무엇을 조사해야 하는가?"를 인간의 판단 호출에서 계산 가능한 답변으로 전환합니다. 이것이 OpenPE를 자율 운영에 적합하게 만드는 것입니다: 에이전트는 자신의 인식론적 한계가 어디에 있는지 알고 있습니다.

### 3. 신뢰 특권 없는 경쟁 인과 DAG

대부분의 인과 추론 도구는 사용자의 인과 모델을 주어진 것으로 받아들이고 그 안에서 파라미터를 추정합니다. OpenPE는 그렇지 않습니다. 0단계는 **여러 경쟁적인 제1원리 DAG**를 생성합니다 — 사용자가 가설을 제공하더라도, 그것은 다른 후보들 중 하나일 뿐이며 특별한 가중치를 받지 않습니다.

**세 가지 입력 모드 — 동일한 처리:**

| 모드 | 사용자가 제공하는 것 | 처리 방식 |
|------|------------------|---------|
| **A** | 질문만 | 프레임워크가 제1원리에서 모든 후보 DAG 생성 |
| **B** | 질문 + 데이터 | 데이터가 구조 발견을 도움; 사용자가 인과 모델 제공 안 함 |
| **C** | 질문 + 가설 | 사용자 가설이 후보 DAG 중 하나 — 프레임워크 생성 대안과 동등하게 경쟁 |

모드 C가 철학적으로 중요합니다. "X가 Y를 유발한다고 믿는다"고 말하는 사용자는 그 믿음을 테스트하는 분석을 받는 것이 아닙니다 — 그 믿음을 대안과 경쟁시키고 어느 것이 살아남는지 보고하는 분석을 받습니다. 이것은 의도적입니다: 가설 확인은 인과 분석이 아닙니다.

**경쟁 DAG는 데이터 수집 전에 평가됩니다.** 0단계의 가설 에이전트는 여러 후보 DAG 구조를 생성하며, 각각에 사전 EP 평가가 있습니다. 이후 데이터 수집 전략은 이들을 구별하도록 설계됩니다 — 선도 가설을 확인하기 위한 것이 아닙니다. 경쟁 설명을 구별할 수 없는 데이터는 표면적으로 주요 가설을 지지하더라도 약한 증거로 표시됩니다.

**대립적 가설 설계는 구체적인 하류 이점이 있습니다.** 3단계 테스트가 DATA_SUPPORTED 분류로 엣지 A→B를 지지할 때, 독자는 물을 수 있습니다: 어떤 대안적 인과 구조가 고려되고 배제되었는가? DISCOVERY.md 산출물은 정확히 이것을 문서화합니다 — 모든 경쟁 DAG, 왜 순위가 낮았는지, 어떤 증거가 그것을 되살릴 수 있는지. 문서화된 대안이 있는 발견은 단일 모델만 테스트하여 얻은 결론보다 강합니다.

**신뢰 특권 없음 규칙은 재귀적으로 적용됩니다.** 분석 중에 원래 후보 DAG에 없는 새로운 인과 메커니즘이 발견되면 중간에 추가할 수 있습니다 — 하지만 다른 엣지와 동일한 가설 생성 및 반박 테스트를 거칩니다. "흥미로운 발견"에서 "확립된 결과"로 가는 빠른 경로는 없습니다.

**왜 중요한가:** 인과 연구에서 가장 흔한 실패 모드는 확인적 분석 — 이미 믿는 것을 찾도록 연구를 설계하는 것입니다. 사용자 가설을 출발점이 아닌 후보로 취급함으로써, OpenPE는 인과 설명을 제안하는 행위와 그것을 테스트하는 행위를 구조적으로 분리합니다.

### 4. 종국 수렴 분류

"원리에서 종국으로"라는 이름은 모든 분석이 어딘가에서 종료된다는 것을 의미합니다. 4단계는 이것을 구체화합니다: 인과 체인에 대한 몬테카를로 예측 후, 프레임워크는 궤적의 *형태*를 네 가지 **종국 카테고리** 중 하나로 분류하고 이름을 붙입니다.

**네 가지 카테고리, 각각 고유한 개입 논리:**

| 카테고리 | 탐지 기준 | 해석 |
|---------|---------|------|
| **견고한 종국** | 시나리오 수렴: 지평선에서 CV < 0.15; 신뢰 구간 축소 | 결과가 가정에 크게 독립적; 방향적 신뢰도 높음 |
| **분기 의존 결과** | 시나리오 급격히 발산: CV > 0.5; 식별 가능한 이진 조건이 궤적 분리 | 결과가 특정 미래 사건에 달려있음 — 분기 지점 이름 짓기, 확률 추정 |
| **균형 종국** | 변화율이 0에 접근; 시나리오가 공통 어트랙터 주변에서 진동 | 시스템이 자연적 균형에 도달 — 균형값과 흡인 영역 식별 |
| **불안정 궤적** | 단조적으로 가속하는 발산; 경계 메커니즘 미탐지 | 폭주 행동 — 결국 억제할 물리적, 규제적, 자원 제약 탐색 |

**분류는 정량적이며, 해석적이지 않습니다.** 예측 지평선에서 시나리오 끝점의 변동계수(CV)가 어떤 카테고리가 적용되는지를 결정합니다. 분석가는 기준 시나리오가 깔끔해 보인다는 이유로 분기 의존 결과를 견고한 종국으로 재분류할 수 없습니다 — 전체 시나리오 포락선의 CV가 결정합니다.

**명명된 시나리오가 분기 조건을 실행 가능하게 만듭니다.** 분기 의존 결과가 탐지되면, 프레임워크는 각 분기를 시나리오로 명명하도록 요구합니다: "시나리오 1 (파라미터 L=104, k=0.19)"이 아니라 "'조기 규제' 시나리오: 채택률이 40% 임계값을 넘기 전에 정부 개입이 성장을 제한함." 확률 추정이 포함된 명명된 시나리오는 의사결정 입력입니다; 파라미터 표는 그렇지 않습니다.

**종국 분류는 EP 감쇠 차트에 입력됩니다** — 모든 OpenPE 보고서의 핵심 그림. 차트의 x축은 현재로부터의 시간이고, y축은 EP 가중 신뢰도이며, 분류는 오른쪽 끝에서 신뢰 구간의 형태를 결정합니다: 견고한 종국의 경우 수렴하는 구간, 분기 의존의 경우 발산하는 구간, 균형 종국의 경우 점근하는 구간.

**왜 중요한가:** 대부분의 예측 프레임워크는 어떤 *유형*의 미래를 예측하는지 특성화하지 않고 결과 분포를 생성합니다. 두 분석 모두 넓은 불확실성 구간을 생성할 수 있습니다 — 하지만 하나는 결과가 진정으로 불확정적이기 때문에(분기 의존), 다른 하나는 단일 어트랙터 주변의 파라미터 불확실성 때문에(견고) 넓습니다. 이것들은 의사결정자에게 완전히 다른 대응을 요구합니다. 종국 분류는 그 구분을 명시적이고 기계 검증 가능하게 만듭니다.

## 입력 모드

| 모드 | 시나리오 | 동작 |
|------|---------|------|
| **A** — 질문만 | 데이터 미제공 | 완전 자율: 가설, 데이터 수집, 품질 게이트 |
| **B** — 질문 + 데이터 | 사용자 데이터셋 제공 | 사용자 데이터는 수집 데이터와 동일한 품질 게이트 통과 |
| **C** — 질문 + 가설 | 사용자 인과 가설 제공 | 사용자 가설은 후보 DAG 중 하나 — 신뢰 특권 없음 |
