---

你提出一个问题或预测。OpenPE 从第一性原理出发追溯到终局——构建因果链、对每条链路进行反驳检验、向前推进直到证据耗尽。返回的结果：每条边都有量化置信度的因果 DAG、每条因果主张的三重反驳检验、分类为稳健 / 分叉依赖 / 均衡 / 不稳定的情景预测，以及将每项发现追溯到数据来源的可机器验证审计链。

整个流水线——假设生成、数据采集、因果检验、预测、报告——全程自主运行。底层原则：解释力沿因果链乘法衰减，框架利用这一衰减决定什么值得研究、什么时候停止。

## 工作原理

```
问题 ─→ 阶段 0：发现 ─→ 阶段 1：策略 ─→ 阶段 2：探索
           │              │              │
       因果 DAG       方法选择        EDA + 图表
       数据采集       EP 评估        分布检查
       质量门控       链规划
           │              │              │
           ▼              ▼              ▼
       阶段 3：分析 ─→ 阶段 4：预测 ─→ 阶段 5：验证
           │              │              │
       因果检验       蒙特卡洛模拟    独立复现
       三重反驳       敏感度分析      EP 传播审计
       EP 传播       终局分类        因果标签审计
           │              │              │
           ▼              ▼              ▼
                  阶段 6：文档
                      │
                  分析报告 (MD + PDF)
                  审计链 (声明 + 数据源 + 逻辑)
                  EP 衰减可视化
```

每个阶段遵循相同的循环：

1. **执行** — 子 agent 生成产物（以规划模式开始）
2. **审查** — 独立审查员将问题分为 A（阻塞）/ B（必须修复）/ C（建议）
3. **迭代** — 应用结构化修复，仲裁者通过 diff 范围验证（非全量重审）
4. **提交** — 状态记录在 STATE.md 中
5. **人工门控** — 阶段 5 之后，人工审批后才进入最终报告

## 四大核心创新

### 1. 解释力 (EP) — 从先验到后验的完整生命周期

多数分析框架将置信度视为二元的："我们相信"或"我们不信"。OpenPE 引入了**解释力 (Explanatory Power)** ——一种连续的、乘法衰减的度量，追踪因果链上解释价值的存留程度，并在整个分析生命周期中持续演化。

**公式：**
```
EP = truth × relevance

truth:     我们对这条因果关系的真实性有多确信？(0–1)
relevance: 这条因果关系解释了结果方差的多大比例？(0–1)
```

**EP 沿因果链乘法衰减。** 对于 A → B → C → D：
```
Joint_EP = EP(A→B) × EP(B→C) × EP(C→D)
```

一条 5 环链、每条边 EP=0.7 的联合 EP 仅为 0.17——刚过软截断阈值。这不是框架的局限，而是间接因果关系中解释力消散的如实呈现。

**截断阈值强制执行停止规则：**

| 联合 EP | 动作 |
|---------|------|
| < 0.05 | **硬截断** — 停止。此链已超出分析视野。 |
| 0.05–0.15 | **软截断** — 仅轻量评估，不展开子链。 |
| > 0.30 | **子链展开** — 值得深入研究，创建子分析。 |

**EP 随分析演化。** 随着证据积累，标签和 truth 值持续更新：

| 阶段 | 分类标签 | truth | 含义 |
|------|---------|-------|------|
| 分析前（先验） | `LITERATURE_SUPPORTED` | 0.70 | 已有文献支持 |
| 分析前（先验） | `THEORIZED` | 0.40 | 机制合理，尚未检验 |
| 分析前（先验） | `SPECULATIVE` | 0.20 | 可能成立，基础薄弱 |
| 阶段 3 后（后验） | `DATA_SUPPORTED` | 0.85 | 通过三重反驳检验 |
| 阶段 3 后（后验） | `CORRELATION` | 0.50 | 统计信号存在，因果性未确立 |
| 阶段 3 后（后验） | `HYPOTHESIZED` | 0.15 | 现有数据无法检验 |
| 阶段 3 后（后验） | `DISPUTED` | 0.10 | 证据内部矛盾 |

先验到后验的转换是机械的，不依赖主观判断：分类表直接映射到 `truth` 值，DATA_SUPPORTED 边的 `truth` 上限为 `min(1.0, max(0.8, prior_truth + 0.2))`。任何分析师都无法绕过这一算术约束。

**DISPUTED 分类用于检测反驳组合中的内部矛盾。** 每条因果边经历三项检验：安慰剂检验（用随机变量替换处理变量；效应应消失）、随机混杂检验（添加随机混杂因子；估计应保持稳定）、数据子集检验（在随机 80% 子集上估计；应保持一致）。安慰剂结果作为因果锚点：

- 如果安慰剂**通过**（效应是处理特异的），那么子集或混杂检验的失败就是矛盾的——真实效应应该是稳定的
- 如果安慰剂**未通过**（效应不是处理特异的），那么子集通过就是矛盾的——非真实的效应不可能完美稳定

```python
# 矛盾检测逻辑（方案 C）
if placebo_passed and (not subset_passed or not cc_passed):
    return "DISPUTED"   # 真实但不稳定——矛盾
if not placebo_passed and subset_passed:
    return "DISPUTED"   # 不真实但完美稳定——矛盾
```

DISPUTED 边路由至人工审查，而非算法猜测。框架拒绝对矛盾证据自动分类。

**为什么重要：** 每条因果论证都携带一个可追溯、可挑战的数字，而不是将不确定性藏在行文措辞里。EP 衰减图——每份 OpenPE 报告的核心图表——将这一传播过程从已知发现到前向预测全程可视化，使认识论的诚实变得看得见。

### 2. 分析视野——EP 截断作为自主研究边界

大多数分析框架对你提供的任何因果结构计算置信度。OpenPE 做的不同：它用 EP 来决定**什么值得研究**。分析视野是联合 EP 的乘法衰减使进一步研究在认识论上不再成立的那个点。

这一区别在实践中至关重要。一个没有停止规则的自主 agent 会无止境地追溯原因——每个结果都有原因，每个原因都有更早的原因，链条永无尽头。分析视野给 agent 提供了一个基于证据结构的、有原则的停止理由，而非任意的深度限制。

**三个硬阈值划定研究边界：**

| 联合 EP | 状态 | 动作 |
|---------|------|------|
| > 0.30 | 值得研究 | 创建子分析；需要完整因果检验 |
| 0.15–0.30 | 边缘区 | 仅轻量评估；记录但不展开 |
| 0.05–0.15 | 软截断 | 简短注记；不允许子链展开 |
| < 0.05 | 硬截断 | 停止。此链已超出分析视野。 |

**视野是涌现出来的，不是预设的。** 它落在证据耗尽的地方——不是参数规定的地方。两条强边（EP=0.85）的因果链比五条弱边更能延伸。深度是证据的结果，不是配置选项。

**这一设计决策直接来自项目名称的含义。** "从原理到终局"不是比喻——它是对分析契约的字面描述。框架从第一性原理出发，向前推进，直到抵达终态（终局）或分析视野（证据耗尽之处）。何时停止不需要人工判断；EP 算术做出这一决定。

**视野与数据质量相互作用。** 如果某数据源被判定为 LOW 质量，它所支撑的边会获得 relevance 惩罚。这可能将一条接近阈值的链拖过软截断边界——分析范围随数据质量下降自动收缩。低质量数据集不会导致失败；它让框架如实反映自己能看多远。

**为什么重要：** 没有停止规则的分析系统不是分析系统——是一个无限产出工作的研究生成器。分析视野将"我们该研究什么"从人工判断变成可计算的答案。这正是 OpenPE 适合自主运行的原因：agent 知道自己的认识论边界在哪里。

### 3. 竞争因果 DAG 与无信任特权

大多数因果推断工具将用户的因果模型当作既定前提，在其中估计参数。OpenPE 不这么做。阶段 0 生成**多个相互竞争的第一性原理 DAG**——如果用户提供了假设，它们也只是众多候选之一，不享有任何特权。

**三种输入模式——同等对待：**

| 模式 | 用户提供什么 | 如何处理 |
|------|------------|---------|
| **A** | 仅问题 | 框架从第一性原理生成所有候选 DAG |
| **B** | 问题 + 数据 | 数据辅助结构发现；用户不提供因果模型 |
| **C** | 问题 + 假设 | 用户假设成为候选 DAG 之一——与框架生成的候选平等竞争 |

模式 C 在哲学上最具意义。一个说"我认为 X 导致 Y"的用户，得到的不是检验这一信念的分析——而是让这一信念与其他候选对抗、并报告哪些存活下来的分析。这是刻意为之的：假设确认不等于因果分析。

**竞争 DAG 在数据采集之前评估。** 阶段 0 的假设 agent 产出多个候选 DAG 结构，每个都有先验 EP 评估。随后的数据采集策略被设计来区分这些候选——而非确认领先假设。无法区分竞争解释的数据即使表面上支持主假设，也会被标记为弱证据。

**对抗性假设设计在下游产生具体收益。** 当阶段 3 检验将 A→B 边支持为 DATA_SUPPORTED 时，读者可以问：哪些替代因果结构被考虑并排除了？DISCOVERY.md 产物精确记录了这些——每个竞争 DAG、为何排名靠后，以及什么证据能让它重新进入候选。有文档记录的替代方案支撑下的发现，比只检验单一模型得出的结论更强。

**无信任特权规则递归适用。** 如果分析中发现了原始候选 DAG 中没有的新因果机制，可以中途添加——但它仍需经历与其他任何边相同的假设生成和反驳检验。从"有趣发现"到"已确立结论"没有捷径。

**为什么重要：** 因果研究中最常见的失败模式是确认性分析——设计研究来找到你已经相信的东西。通过将用户假设视为候选而非出发点，OpenPE 在结构上将提出因果解释与检验因果解释这两个行为分离开来。

### 4. 终局收敛分类

"从原理到终局"这个名字意味着每次分析都会在某处终止。阶段 4 将这一点落实：在对因果链进行蒙特卡洛预测后，框架将轨迹的*形态*分类为四种**终局类别**之一并命名。

**四个类别，各有不同的干预逻辑：**

| 类别 | 判定标准 | 含义 |
|------|---------|------|
| **稳健终局** | 情景收敛：视野处 CV < 0.15；置信带收窄 | 结果基本不依赖假设；方向置信度高 |
| **分叉依赖结果** | 情景急剧发散：CV > 0.5；可识别的二元条件分隔轨迹 | 结果取决于某个具体的未来事件——命名分叉，估计其概率 |
| **均衡终局** | 变化率趋近于零；情景在共同吸引子附近振荡 | 系统达到自然平衡——识别均衡值和吸引域 |
| **不稳定轨迹** | 单调加速发散；未检测到边界机制 | 失控行为——寻找最终会约束它的物理、监管或资源边界 |

**分类是定量的，不是解释性的。** 预测视野处情景端点的变异系数 (CV) 决定适用哪个类别。分析师不能因为基准情景看起来干净就将分叉依赖结果重新分类为稳健终局——完整情景包络的 CV 才是判据。

**命名情景让分叉条件具有可操作性。** 当检测到分叉依赖结果时，框架要求将每个分叉命名为一个情景：不是"情景 1（参数 L=104, k=0.19）"，而是"'早期监管'情景：政府干预在采纳率突破 40% 门槛之前约束了增长"。带有概率估计的命名情景是决策输入；参数表不是。

**终局分类为 EP 衰减图提供输入**——每份 OpenPE 报告的核心图表。图表横轴是距今时间，纵轴是 EP 加权置信度，终局分类决定置信带在右侧边缘的形态：稳健终局对应收敛带，分叉依赖对应发散带，均衡终局对应渐近带。

**为什么重要：** 大多数预测框架产出结果分布，却不描述它们预测的是*哪种类型*的未来。两个分析都可能产出宽不确定带——但一个宽是因为结果本质上不确定（分叉依赖），另一个宽是因为单一吸引子周围的参数不确定性（稳健）。这两种情况对决策者意味着完全不同的应对方式。终局分类让这一区别变得明确且可机器验证。

## 输入模式

| 模式 | 场景 | 行为 |
|------|------|------|
| **A** — 仅问题 | 未提供数据 | 全自主：假设生成、数据采集、质量门控 |
| **B** — 问题 + 数据 | 用户提供数据集 | 用户数据通过与采集数据相同的质量门控 |
